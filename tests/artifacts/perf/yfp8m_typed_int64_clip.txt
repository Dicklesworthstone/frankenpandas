br-frankenpandas-yfp8m — Typed Int64 fast path for Series/DataFrame.clip
========================================================================

Int64 vein (after z0dze i64.gt 154x, drhsx i64.diff 49x). Series.clip's typed fast
path (0zp9e) was Float64-only; an all-valid Int64 column with integer-valued bounds
is the dtype-preserving (Int64-output) case, but it ran the per-element Scalar
values() loop AND a separate has_nulls scan that materialized the whole Vec<Scalar>.

LEVER (one): when the column is all-valid Int64 with integer-valued bounds, clamp
each i64 to the effective bounds cast to i64 into a typed Int64 buffer
(from_i64_values), and prove no-nulls via validity().all() instead of the
materializing has_nulls scan.

ISOMORPHISM:

    integer-valued via is_integer_bound + no nulls), so the output is Int64, as
