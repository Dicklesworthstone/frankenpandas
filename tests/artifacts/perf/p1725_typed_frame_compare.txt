eydcr third-wave op. A widened gap_hunt probe (comparison/where/mask/isin/scalar

the f64 + validity slices (no align, no input Scalar materialization, no per-element
to_f64). One fast path serves all six comparison ops.

ISOMORPHISM:
  - A pair is missing iff EITHER side is missing — validity clear OR a valid-bit
    NaN (which Scalar::is_missing treats as missing), matching the general path's
    `if lv.is_missing() || rv.is_missing()` -> Null(NullKind::Null).
  - Otherwise `l <op> r` over the identical present f64s — the same numeric arm the
    general path takes via to_f64 (Float64 operands never reach the string arm).
  - Output column is built the same way: from_values of the [Bool / Null(Null)]
    vector when any pair is missing; from_bool_values when none is (an all-valid
    Bool that materializes to the same Bool scalars).
  - Gated to identical index + identical column order + all-Float64 both sides; any
    other shape/dtype keeps the exact align + Scalar path.
