A sixth gap_hunt probe wave (mode/sem/prod) found mode 56ms (the biggest fresh
gap). Series::mode_with_dropna (which DataFrame.mode calls per column) already had

ISOMORPHISM (produces the identical (value, count) SET, which is all that matters —
the mode result is sorted + max-count-filtered downstream, so tally order is moot):
  - A value is tallied iff its materialized Scalar is Float64(finite) — validity set
    AND datum not NaN. Its key is FloatBits of the -0.0 -> +0.0 normalized datum,
    exactly scalar_key_allow_missing's Float64 arm (so +0.0/-0.0 collapse to one
    key); the stored value is the FIRST-seen original Float64, matching the
    ScalarKey path's value.clone() of the first occurrence.
  - Gated to dropna OR all-valid: the only missing-value bucket arises with
    !dropna AND missing present, which is left to the ScalarKey path (so the
    materialized first-seen missing Scalar is reproduced exactly there). With
    dropna (the df.mode() default) all missing are skipped — identical to the
    ScalarKey path's `if dropna && is_missing { continue }`.
  - Int64 (direct-address) and Utf8 (byte-span) fast paths are untouched.
  - Golden sha256 (gap_hunt golden: full elementwise/reduction family + mode over a
    period-9973 repeated-value frame (real modes, count>1), its nullable variant,
    and an all-distinct frame (every value a mode), n=5000-20000) UNCHANGED:
