**Two giant blast zones**: SeriesGroupBy (20% covered) and MultiIndex (19% covered). Combined missing surface there alone = 151 methods.

**Five Index variants** (DatetimeIndex, TimedeltaIndex, PeriodIndex, RangeIndex, CategoricalIndex) don't exist as dedicated types in fp-index — total 624 methods. Some functionality lives on the generic `Index` or on `DatetimeAccessor`, but the type-level public surface is missing.

**No plotting** (plot/hist/boxplot on DataFrame/Series/GroupBy) — likely intentional out-of-scope but should be a documented decision, not a silent gap.

## Coverage Methodology

- **Upstream surface**: `artifacts/pandas_api_listing.json` (pandas 2.2.3, generated by `scripts/gen_pandas_api_listing.py`). 15 classes covered.
- **Implementation surface**: scanned `crates/fp-frame/src/lib.rs` and `crates/fp-index/src/lib.rs` for `pub fn` inside `impl <Class>` and `impl <Class><'_>` blocks; matched class names by `fp-impl → pandas-class` mapping (`Ewm → ExponentialMovingWindow`, `Resample → Resampler`, etc.).
- **Coverage** = `|impl_methods ∩ pandas_methods| / |pandas_methods|`. Implementation-only methods (e.g., `add_df` on DataFrame instead of pandas' `add`) count as MISSING because pandas-parity callers use the pandas name.
- **Naming-mismatch caveat**: 8 DataFrame arithmetic ops are implemented as `<op>_df` and `<op>_scalar` instead of pandas' `<op>(other)`. Same for Series in 4 cases (`add_fill` vs pandas `add(other, fill_value=...)`). These show as "missing" in the table; the bead `reality-2x9rj` (DataFrame arithmetic alias matrix) tracks the rename/re-export.

## Per-Class Gap Detail

### DataFrame — 73 missing

**Arithmetic / comparison aliases (20 methods)** — pandas style:
`add, sub, mul, div, divide, multiply, subtract, truediv, floordiv, mod, pow, radd, rsub, rmul, rdiv, rtruediv, rfloordiv, rmod, rmul, rpow` plus `eq, ne, lt, gt, le, ge`. Today implemented as `add_df`/`sub_df`/`mul_df`/`div_df` etc. (12 variants).

**IO method-on-class wrappers (16 methods)** — pandas users call `df.to_X()`:
`to_clipboard, to_excel, to_feather, to_gbq, to_hdf, to_orc, to_parquet, to_period, to_pickle, to_sql, to_stata, to_string, to_timestamp, to_xarray, to_xml`. Free functions exist in fp-io for excel/feather/parquet/sql; the rest are absent entirely.

**Method-on-class delegators to existing functions (8)**:
`eval, query, filter, map, merge, join, where, align`. Backing logic exists in fp-expr and fp-join — just not exposed as DataFrame methods.

**Timezone (2)**: `tz_convert, tz_localize` (DatetimeAccessor exists for Series; DataFrame method missing).

**Reshape / metadata (12)**: `T` (transpose alias), `values, attrs, flags, set_flags, asfreq, first, last, backfill, pad, isetitem, get, bool, reindex_like, rename, rename_axis, style, sparse, product`.

**Plotting (3)**: `plot, hist, boxplot`. Probably out-of-scope; needs a documented decision.

### SeriesGroupBy — 55 missing (CRITICAL)

Only 14/69 methods implemented. Almost everything pandas users routinely call is missing:

- **Reductions**: `all, any, idxmax, idxmin, nlargest, nsmallest, quantile, sem, skew, nunique, value_counts, unique`
- **Cumulative**: `cumcount, cummax, cummin, cumprod, cumsum`
- **Apply / transform**: `apply, transform, agg, aggregate, pipe, filter`
- **Slicing**: `head, tail, take, sample, nth, get_group`
- **Missing-value**: `bfill, ffill, fillna`
- **Time / window**: `rolling, expanding, ewm, resample, ohlc`
- **Diffs**: `diff, shift, pct_change`
- **Iteration / introspection**: `keys, groups, grouper, indices, level, ndim, ngroup, ngroups, dtype, is_monotonic_decreasing, is_monotonic_increasing`
- **Plotting**: `plot, hist`

This is the single largest functional gap in the project.

### MultiIndex — 96 missing (CRITICAL)

Only 22/118 methods. Almost the entire pandas MultiIndex surface is absent:
- All set operations (`difference, intersection, symmetric_difference, union`)
- All iteration / restructure (`append, delete, insert, drop, drop_duplicates, dropna, fillna, repeat, putmask, set_codes, set_levels, sortlevel, sort_values, swaplevel, remove_unused_levels`)
- All conversion (`to_frame, to_list, to_numpy, to_series, tolist, array, values, ravel, view, T, transpose, format`)
- All search / lookup (`asof, asof_locs, get_indexer, get_indexer_for, get_locs, get_slice_bound, searchsorted, slice_indexer`)
- All reductions (`all, any, argmax, argmin, max, min, nunique, value_counts`)
- All metadata (`name, dtype, dtypes, codes, levels, levshape, hasnans, holds_integer, infer_objects, inferred_type, item, ndim, nbytes, memory_usage, shape, size, empty`)
- All comparisons (`equals, equal_levels, identical, is_, is_boolean, is_categorical, is_floating, is_integer, is_interval, is_numeric, is_object, isna, isnull, notna, notnull`)
- All restructure (`copy, factorize, fillna, groupby, join, map, rename, reindex, shift, sort, str, truncate, where, from_frame, unique`)

### Index variants — 624 methods across 5 dedicated types (CRITICAL)

`fp-index` exposes one generic `Index` type. Pandas has 5 specialized subclasses with ~100-150 dedicated methods each:

| Type | Pandas methods |
|---|---|
| DatetimeIndex | 154 |
| MultiIndex | 118 (separate gap above) |
| PeriodIndex | 132 |
| TimedeltaIndex | 121 |
| CategoricalIndex | 112 |
| RangeIndex | 105 |

A `DatetimeAccessor` type exists on Series for date-time methods (`.dt.year`, etc.). But there's no `DatetimeIndex` type, so `Index.year` returns nothing. Same for the other variants.

This is a decision point: (a) implement the 5 variants as separate types (large), or (b) overload `Index` with type-tag dispatch (compromise), or (c) document that fp-frame treats Index variants as accessors only and re-shape the parity story (smallest).
